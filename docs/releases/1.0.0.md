# 1.0.0 Release Notes

## English

### Changes

- Added manual active-note sync to Notion with immediate progress notices.
- Added manual pull from Notion back to the active Obsidian note.
- Added property list fetching for each database profile.
- Added Notion property dropdowns for mapping setup in settings.
- Added English and Simplified Chinese UI copy.
- Added GitHub release build support with `npm run build:release` and release workflow automation.

### Fixes

- Fixed `sync active note database` when only one database is configured.
- Fixed missing local property updates after remote page creation or update.
- Fixed Notion `fetch` transport issues in the Obsidian runtime by using `requestUrl` first and sanitizing the fallback `fetch`.
- Fixed date property serialization for Notion `date` fields.
- Improved sync diagnostics so skipped states and transport failures are easier to identify.

## 简体中文

### 变更

- 新增手动将当前笔记同步到 Notion，并在开始时立即显示进度提示。
- 新增从 Notion 手动拉取回当前 Obsidian 笔记的能力。
- 新增数据库配置的属性列表拉取功能。
- 设置页中的属性映射改为使用 Notion 属性下拉框。
- 新增英文和简体中文 UI 文案支持。
- 新增 GitHub Release 构建支持，包括 `npm run build:release` 和自动发布工作流。

### 修复

- 修复仅配置一个数据库时 `sync active note database` 不可用的问题。
- 修复远端页面创建或更新后，本地属性没有同步回 frontmatter 的问题。
- 修复 Obsidian 运行时下 Notion `fetch` transport 不稳定的问题，优先使用 `requestUrl`，并清洗回退 `fetch` 请求。
- 修复 Notion `date` 属性的日期序列化格式错误。
- 改进同步诊断，让跳过原因和底层传输失败更容易定位。
